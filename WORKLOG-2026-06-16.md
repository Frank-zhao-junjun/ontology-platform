### 2. 创建 `.coze` 配置文件

**文件**: `.coze` (项目根目录)

**配置内容**:
```toml
[project]
requires = ["java-21", "maven-3.8"]
entrypoint = "ontology-api/src/main/java/com/ontology/platform/api/OntologyPlatformApplication.java"

[dev]
build = ["sh", "-c", "mvn clean compile -DskipTests -q"]
run = ["sh", "-c", "mvn -pl ontology-api spring-boot:run -DskipTests -Dspring-boot.run.profiles=dev"]

[deploy]
build = ["sh", "-c", "mvn clean package -DskipTests -q"]
run = ["sh", "-c", "java -Dserver.port=${DEPLOY_RUN_PORT:-8080} -Dspring.profiles.active=dev -jar ontology-api/target/ontology-api-1.0.0-SNAPSHOT.jar"]
```

**功能**:
- 支持 `coze dev` 一键启动开发环境
- 支持 `coze build` + `coze start` 生产部署
- 自动绑定 `${DEPLOY_RUN_PORT}` 环境变量
- 默认使用 `dev` profile（PostgreSQL + Flyway + MyBatis-Plus；需先 `docker compose up postgres redis`）

### 3. 创建 `application-dev.yml` 开发配置

**文件**: `ontology-api/src/main/resources/application-dev.yml`

**核心配置**（2026-06-16 修订）:
| 配置项 | 值 | 说明 |
|--------|-----|------|
| 数据库 | PostgreSQL（继承 `application.yml`） | Flyway 脚本含 PG 专有语法，**不支持 H2** |
| 端口 | `${DEPLOY_RUN_PORT:-8080}` | 自动绑定环境变量 |
| JPA ddl-auto | `none` | 建表由 Flyway 负责，非 Hibernate |
| Flyway | `enabled: true` | 与 MyBatis-Plus 配套 |
| Redis | localhost:6379 | Phase 2 Job / 限流 / Webhook 依赖 |

**前置**:
```bash
cd docker && docker compose up -d postgres redis
```
