### 2. 创建 `.coze` 配置文件

**文件**: `.coze` (项目根目录)

**配置内容**:
```toml
[project]
requires = ["java-21", "maven-3.8"]
entrypoint = "Ontology-platform/ontology-api/src/main/java/com/ontology/platform/OntologyApiApplication.java"

[dev]
build = ["sh", "-c", "cd Ontology-platform && mvn clean compile -DskipTests -q"]
run = ["sh", "-c", "cd Ontology-platform && export DEPLOY_RUN_PORT=${DEPLOY_RUN_PORT:-8080} && mvn spring-boot:run -pl ontology-api -am -DskipTests -Dspring-boot.run.jvmArguments=\"-Dserver.port=${DEPLOY_RUN_PORT}\" -Dspring.profiles.active=dev"]

[deploy]
build = ["sh", "-c", "cd Ontology-platform && mvn clean package -DskipTests -q"]
run = ["sh", "-c", "cd Ontology-platform && export DEPLOY_RUN_PORT=${DEPLOY_RUN_PORT:-8080} && java -Dserver.port=${DEPLOY_RUN_PORT} -Dspring.profiles.active=dev -jar ontology-api/target/ontology-api-1.0.0-SNAPSHOT.jar"]
```

**功能**:
- 支持 `coze dev` 一键启动开发环境
- 支持 `coze build` + `coze start` 生产部署
- 自动绑定 `${DEPLOY_RUN_PORT}` 环境变量
- 默认使用 `dev` 配置文件（H2内存数据库）

### 3. 创建 `application-dev.yml` 开发配置

**文件**: `ontology-api/src/main/resources/application-dev.yml`

**核心配置**:
| 配置项 | 值 | 说明 |
|--------|-----|------|
| 数据库 | H2 (内存模式) | 无需 PostgreSQL 即可启动 |
| 端口 | `${DEPLOY_RUN_PORT:-8080}` | 自动绑定环境变量 |
| JPA ddl-auto | `create-drop` | 自动建表 |
| Flyway | `enabled: false` | 禁用迁移（H2自动建表） |
| H2 Console | `enabled: true` | 路径: `/api/h2-console` |

**访问信息**:
- H2 Console: `http://localhost:8080/api/h2-console`
- JDBC URL: `jdbc:h2:mem:ontology`
- Username: `sa`
- Password: (空)

## 五、快速启动命令

### 开发环境（推荐）
```bash
# 使用 Coze CLI（H2内存数据库，无需Docker）
coze dev

# 或使用 Maven
mvn spring-boot:run -Dspring.profiles.active=dev
```

### 生产环境
```bash
# 使用 Coze CLI
coze build
coze start

# 或使用 Docker Compose（需 PostgreSQL + Redis）
cd docker
docker compose up -d
```
