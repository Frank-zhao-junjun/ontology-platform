## 2. 模型范围对比

| 维度 | 项目1（当前） | 项目2（当前） | 差距 |
|------|--------------|--------------|------|
| 核心模型 | 数据/行为/规则/事件/**流程** 5 元模型 | 行为/事件/EPC 步骤部分落库 | 缺 **Process/Orchestration** 一等存储 |
| 平台模型 | 治理/指标/数据源/元数据/主数据 | 治理 + 数据源（Manifest 内） | 缺 **Metrics、Metadata、MasterData** |
| 组织模型 | Department + Position + PositionResponsibility + HR 同步 | 无 | **全新域**，需表 + API + 治理链路 |
| EPC | EpcChain/Node/Edge + 12 模型 refs，71 条校验 | `epc_step` 扁平表 | 需 **图结构 + 全域关联** |
| Lifecycle | State/Transition/Action 增强 + EntityLifecycle 聚合 | `state_machine` / `action_definition` 基础字段 | 字段与语义 **大幅扩展** |
| Semantic Layer | Intent/SlotFilling/BusinessTerm/… 9 子模型 | MCP `resolve_intent` 占位 | 需 **语义层持久化 + MCP 对接** |
| 实体角色 | `aggregate_root` / `child_entity` | `aggregate_root` / `entity` | **枚举不一致** |
| 业务归属 | `businessScenarioId` 强制 | boundedContext / ontology 级 | 需 **场景维度** |
| 属性契约 | `metadataTemplateId`、`referenceKind`、`masterDataType`… | Property 简单 dataType | **属性模型不对齐** |
| 校验规则 | V01–V11 + VE×17 + VM×39 + VX×15 + V-LC×15 + V-AS×15 + 组织/Excel | ManifestValidator **仅 V01–V11** | 校验体系需 **10× 扩展** |
| 交换格式 | 项目 JSON + Excel 8 Sheet + 未来版本发布包 | `OntologyManifest` YAML v1 | 需 **Manifest v2 或新 apiVersion** |
| 导出 API | `GET /api/agent-semantic-layer`、`GET /api/entity-lifecycle` | 无对应 REST | 平台需 **镜像接收或编译** |

## 3. 交换契约：必须重定

### 3.1 当前项目2 契约（过时风险）

- [`ontology-manifest-spec.md`](./ontology-manifest-spec.md) — `apiVersion: ontology.platform/v1`
- 样例 [`examples/manufacturing-manifest.yaml`](./examples/manufacturing-manifest.yaml)
- 校验：11 条，与项目1 的 71+ 规则不对等

### 3.2 项目1 实际产出形态

1. **Zustand 持久化 JSON**（`OntologyProject` 全量）
2. **Excel 导入** — 8 Sheet（实体/属性/关系/状态机/规则/事件/部门/岗位）
3. **版本对象** — `ProjectVersion`（`draft` → `pending_review` → `published`）
4. **分模块导出 API** — EntityLifecycle、AgentSemanticLayer JSON
5. **规划中** — 版本发布代码包（Flask/React），与项目2 的 Spring/MCP 路径不同

**结论**：不能假设项目1 仍只导出旧版 YAML。对接采用 **方案 A**：`spec.project` **原样嵌入** `OntologyProject`（`dataModel`、`behaviorModel`、`agentSemanticLayer` 等字段名与 `ontology.ts` 1:1）。

```
ontology.platform/v2
├── metadata（交换包元信息）
└── spec
    ├── project: OntologyProject     ← 与 ontology.ts 1:1
    ├── extensions?（metadataList, masterData）
    ├── epc?（EpcChain 目标态）
    ├── lifecycle?（EntityLifecycle 编译）
    └── validation?
```
