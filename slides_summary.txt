  - 'MCP 分发机制 ★ 推荐'
  - '本体 → 编译为 Agent Manifest → MCP Server 加载 → Agent 通过 MCP 工具白名单调用'
  - 'MCP 是开放标准协议，适用于 Dify / Coze / LangChain 等所有主流AI平台'
  - 'Agent 工具列表由 Manifest 动态编译 —— 不同本体暴露不同工具，完全隔离'
  - '硬约束 100%：Agent 无 SQL/API/DB 直接访问权，结构上不可绕过'
  - '支持三通道纵深防御：意图识别 → 白名单入口 → 校验层拦截'
  - '版本化管理：Manifest 版本与本体版本严格对应，快照不可变'
  - '平台内置 3 个固定工具 + N 个动态编译工具（由本体定义自动生成）'
  - 'Skill 方式 △ 受限'
  - 'Skill 是提示词层面的约束——依赖 LLM "自觉遵守"，无法保证 100%'
  - '缺乏结构化的工具白名单——LLM 理论上可绕过 Skill 直接操作数据'
  - '无法做硬约束——没有 MCP 工具清单编译和白名单拦截机制'
  - '不适合企业级安全合规要求——审计链不完整，执行不可追溯'
  - '结论'
  - 'MCP 是唯一能保证 100% Agent 合规的分发方式'
  - '核心原则：'
  - '"不让 Agent 直接接触数据基础设施——'
  - '100% 遵守是结构必然而非意愿问题"'
