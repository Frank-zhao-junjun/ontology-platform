## 二、代码质量问题清单

### 2.1 🔴 高优先级问题

#### 问题 1: 发票数据使用内存存储（数据丢失风险）

**文件**: `src/app/api/invoices/route.ts`

```typescript
// 发票数据存储（内存）
let invoices: Record<string, Record<string, unknown>> = {};
```

**风险**: 
- 服务重启后数据全部丢失
- 无法水平扩展（多实例数据不一致）
- 无持久化，无备份

**建议**: 使用 Supabase/PostgreSQL 存储

---

#### 问题 2: 用户认证机制不安全

**文件**: `src/lib/api-permission.ts` (第 25-50 行)

```typescript
export function getUserIdFromRequest(request: NextRequest): string | null {
  // ...
  // 方式 3：从 URL 参数获取（仅用于演示）
  const userIdParam = searchParams.get('user_id');
  if (userIdParam) {
    return userIdParam;
  }
  return null;
}
```

**风险**:
- 用户可通过修改 URL 参数冒充其他用户
- 缺少 JWT/Session 验证
- 权限检查形同虚设

**建议**: 
- 移除 URL 参数获取用户 ID 的方式
- 实现完整的 JWT 验证
- 使用 NextAuth.js 或 Supabase Auth

---

#### 问题 3: 发票 ID 生成逻辑缺陷

**文件**: `src/app/api/invoices/route.ts`

```typescript
let invoiceCounter = 0;

function generateInvoiceNumber(): string {
  const date = new Date();
  const y = date.getFullYear();
  const m = String(date.getMonth() + 1).padStart(2, '0');
  const d = String(date.getDate()).padStart(2, '0');
  invoiceCounter++;
  return `INV-${y}${m}${d}-${String(invoiceCounter).padStart(4, '0')}`;
}
```

**风险**:
- 服务重启后 counter 归零，会产生重复 ID
- 并发情况下会重复

**建议**: 使用数据库自增或 UUID

#### 问题 6: ID 生成使用 Math.random()

**多处出现**


**风险**: Math.random() 不是加密安全的随机数

**建议**: 使用 nanoid() 或 crypto.randomUUID()
