## 3. 遗留问题清单

### 3.1 阻塞开发（必须解决）

| 问题 | 优先级 | 解决方案 | 预计工时 |
|------|--------|----------|----------|
| **filter参数SQL注入风险** | P0 | 使用DSL解析器替代字符串拼接 | 1-2天 |

**修复方案建议**：
```java
// 禁止：直接拼接（当前方案）
String sql = "SELECT * FROM orders WHERE " + filter;

// 推荐：DSL解析器
FilterExpression expr = FilterParser.parse(filter);
Query query = QueryBuilder.build(expr);
```

## 4. 开发启动建议

### 4.1 首批开发任务建议

```
Phase 1: 基础设施（第1-2周）
├── 1. 环境搭建（开发/测试环境）
├── 2. 修复filter参数SQL注入风险（P0）
├── 3. 搭建CI/CD流水线
└── 4. 技术方案评审（API安全、缓存策略）

Phase 2: 核心功能（第3-6周）
├── 1. 本体管理服务（CRUD + 发布）
├── 2. 查询服务（过滤、分页、关联查询）
├── 3. API网关（认证、限流、路由）
└── 4. 基础前端（本体设计器）

Phase 3: 扩展功能（第7-8周）
├── 1. 动作执行服务
├── 2. 权限管理
├── 3. 图谱可视化（限制节点数）
└── 4. 审计日志

Phase 4: 集成与交付（第9-10周）
├── 1. 前后端集成
├── 2. 部署验证
├── 3. 端到端测试
└── 4. 文档完善
```

### 4.2 风险监控点

| 风险点 | 概率 | 影响 | 监控方式 | 应对措施 |
|--------|------|------|----------|----------|
| MVP范围蔓延 | 高 | 高 | 周会Scope Review | 严格执行MVP边界 |
| 图数据库性能不足 | 中 | 中 | 性能测试 | 限制查询深度+添加索引 |
| 实施难度超出预期 | 中 | 中 | POC验证 | 提前进行Demo演示 |
| Apache AGE兼容性 | 低 | 中 | 单测+集成测 | 保持与社区联系 |

### 4.3 成功标准（MVP阶段）

建议在Kickoff Meeting时与产品对齐：

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 功能完成率 | ≥90% P0功能 | MVP功能范围覆盖率 |
| API响应时间 | P99 < 500ms | 普通查询性能 |
| 缺陷密度 | < 5个/千行代码 | 代码质量 |
| 文档覆盖率 | 100%核心API | 文档完整性 |
