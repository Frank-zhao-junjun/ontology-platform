### P1-1: JobPoller — 异步执行引擎 (3 files)

| 文件 | 说明 |
|------|------|
| `JobHandler.java` | 接口: `supportedJobType()` + `execute(payload, tenantId, agentId)` |
| `ImportJobHandler.java` | 实现 `import.execute` → 映射 payload → `ImportService.executeImport()` |
| `JobPoller.java` | `@Scheduled(fixedDelay=2000)` 轮询 Redis, 每次最多 3 个 job, 自动重试 |

**JobPoller 执行流程**:
```
dequeue → RUNNING → handler.execute() → COMPLETED → webhook
                            ↓
                        FAILED → retry (max 3) or FAILED → webhook
```

### P1-2: Webhook 回调链 (集成到 JobPoller)

JobPoller 在 `completeJob()` / `failJob()` 中自动:
1. 查询 `webhook_subscription` (按 eventType: `job.completed` / `job.failed`)
2. 构建 event payload `{eventType, jobId, timestamp, result}`
3. 调用 `WebhookDispatcher.dispatch(url, secret, event)` → HMAC-SHA256 签名投递

```
JobPoller
  ├─ completeJob → webhookMapper.selectByEventType("job.completed")
  │                  → webhookDispatcher.dispatch(url, secret, event)
  └─ failJob    → webhookMapper.selectByEventType("job.failed")
                    → webhookDispatcher.dispatch(url, secret, event)
```

#### 2. pom.xml — 添加 `-parameters` 编译器标志
- **问题**: `@PathVariable String id` 运行时参数名丢失 (arg0)，Spring 抛出 `IllegalArgumentException`
- **修复**: `<compilerArgs>` 中追加 `<arg>-parameters</arg>`

#### 3. OntologyControllerTest — `$.success` → `$.code`
- **问题**: `ApiResponse` 无 `success` 字段，实际结构为 `{code, message, data, meta}`
- **修复**: 成功断言 `$.success(true)` → `$.code(0)`；失败断言 `$.success(false)` → `$.code(is(not(0)))`
